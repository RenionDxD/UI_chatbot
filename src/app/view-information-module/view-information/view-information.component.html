ChatBot

El módulo de chat utiliza respuestas mejoradas generadas por GPT para ofrecer interacciones más amigables y corteses. Las respuestas se adaptan para brindar una experiencia de usuario más personalizada y satisfactoria.

Subir Datos

El módulo permite cargar dos archivos que contienen datos esenciales del chat: un archivo de prompts y otro en formato XLSX con las preguntas y respuestas..

Carga de Documentos en el Módulo

El módulo de subir datos requiere la carga de dos documentos. Se proporcionará un botón para cargar ambos archivos, aunque es suficiente cargar al menos uno. La acción de carga se inicia a través de un botón que brinda acceso a las carpetas del dispositivo para seleccionar la información requerida. Después de la selección, se desplegará una confirmación para proceder con el envío. Un texto informativo mostrará el resultado de la operación debajo de los módulos, indicando éxito en la verificación o la presencia de errores.

Promps txt
  1. El primer archivo debe estar en formato .txt.
  2. Se requieren exactamente 4 líneas, correspondientes a 4 promps
  3. Los nombres de las variables deben coincidir con las columnas del archivo XLSX.
  4. Es necesario separar las variables y los textos de los promps mediante el símbolo =.
  5. El texto final debe terminar con un ":" para indicar la separación entre el promp y el texto a reformular por el modelo GPT.
archivo xlsx
  1. Deberá ser un archivo con formato XLSX.
  2. Las columnas deben estar al inicio de la hoja para una correcta lectura del sistema; cualquier desplazamiento podría interferir con la lectura.
  3. Se deben verificar la existencia de las columnas principales: detalle y solución, que el chat leerá.
  4. Se deben confirmar las 4 columnas de texto: código, usuario, soporte, y solicitud cliente (en ese orden).
  5. En las columnas código, usuario, soporte y solicitud cliente, asegúrate de colocar una 'X' mayúscula según la dirección del usuario.
  6. Verifica la ortografía en los nombres de las columnas, ya que errores podrían causar fallos o impedir la lectura de la información.
  7. Si es posible, completa cada uno de los datos, utilizando (vacio) si no hay respuesta para algún campo.
  8. Asegúrate de incluir solo datos pertinentes a las columnas; información irrelevante podría ocasionar fallos en el sistema.

Creacion de modelo

Generacion de modelo de ChatBot

En este modulo, se ha habilitado un único botón cuyo propósito es emplear la totalidad de la información incluida en los archivos TXT y XLSX con el fin de ejecutar el procesamiento de datos correspondiente. Tras la conclusión del procesamiento, se generará un informe exhaustivo que documentará las operaciones realizadas. En caso de detectarse algún fallo, se ofrecerá la posibilidad de identificar y localizar el origen del problema.
